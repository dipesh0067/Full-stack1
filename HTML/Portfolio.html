Section 01 — Markup

HTML – Structure of Every Page

HTML (HyperText Markup Language) is the skeleton of every webpage. It uses tags to label content so the browser knows what each piece is and how to display it.

📄
<h1>–<h6>

Heading hierarchy for page structure and SEO

🔗
<a>

Anchor links — connects pages together across the web

📝
<form>

Collects user input and sends data to the backend

🖼️
<img>

Embeds images with alt text for accessibility

index.html
<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>My ePortfolio</title> </head> <body> <h1>Dipesh Yadav</h1> <p>CET138 student.</p> <ul> <li>HTML & CSS</li> <li>Bootstrap</li> <li>JavaScript</li> </ul> <img src="photo.jpg" alt="Profile"> <form> <input type="text" placeholder="Name"> <button type="submit">Send</button> </form> </body> </html>

Section 02 — Styling

CSS – Making Pages Look Good

CSS (Cascading Style Sheets) controls how HTML elements look — colours, fonts, spacing, and layout. Without CSS every page would be plain black text on a white background.

🔴
Inline CSS

Written directly inside an HTML tag. Quick but messy — hard to maintain.

<p style="color:red;">Text</p>
🟡
Internal CSS

Written in a <style> tag in the HTML head. Fine for small single-page projects.

<style> p { color: blue; } </style>
🟢
External CSS ✓

A separate .css file linked via <link>. One stylesheet controls the whole site. Used in this portfolio.

<link rel="stylesheet" href="CSS/index.css">
CSS/index.css — key rules from this site
/* CSS Variables */ :root { --primary: #16a34a; --dark: #0f172a; --accent: #4ade80; --text: #e2e8f0; } /* Body styling */ body { font-family: 'Segoe UI', Arial; background: var(--dark); color: var(--text); } /* Box model */ .card-custom { background: #1e293b; border-radius: 10px; padding: 24px; border: 1px solid #334155; } /* Flexbox layout */ .css-demo-profile { display: flex; align-items: center; gap: 20px; } /* Hover effect */ .card-custom:hover { border-color: var(--primary); transform: translateY(-4px); }

Section 03 — Framework

Bootstrap – Build Faster

Bootstrap is a free CSS framework that provides a 12-column responsive grid and ready-made components — navbar, cards, buttons — added with one CDN link.

index.html — Bootstrap CDN setup
<!-- Add to <head> --> <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/ bootstrap@5.3.0/dist/css/ bootstrap.min.css"> <!-- Add before </body> --> <script src="https://cdn.jsdelivr.net/ npm/bootstrap@5.3.0/dist/js/ bootstrap.bundle.min.js"> </script> <!-- Responsive 3-column grid --> <div class="container"> <div class="row"> <div class="col-md-4">Card 1</div> <div class="col-md-4">Card 2</div> <div class="col-md-4">Card 3</div> </div> </div>
Why Bootstrap saves time

A responsive navbar from scratch takes an hour — CSS for desktop, media queries for mobile, JS for the hamburger. Bootstrap does all this with 10 lines of HTML and the right class names. No extra code needed.

Section 05 — Architecture

Full Stack – How It All Connects

Full stack development covers every layer of a web app — the frontend users see, the backend server logic, and the database that stores data permanently.

🖥️
Frontend

What users see. Built with HTML, CSS, JavaScript and Bootstrap. Runs in the browser.

⚙️
Backend

Server-side logic. Handles requests, authentication, and business rules using Node.js or Python.

🗄️
Database

Stores all data permanently. MySQL for structured, MongoDB for flexible document storage.
